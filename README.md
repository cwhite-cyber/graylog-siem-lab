# Graylog SIEM LabDeployment of a Graylog SIEM stack (MongoDB + OpenSearch + Graylog server) as a new VM in an existing segmented home lab (pfSense firewall, LibreNMS monitoring, Kali). This project closes the loop between the firewall and the monitoring stack: pfSense enforces policy, LibreNMS watches network health, and Graylog aggregates and searches logs from across the lab.> Status: In progress - build log and troubleshooting notes below are updated as I go. Only errors I actually hit during this build are documented; nothing here is copy-pasted from someone else's writeup.## Why this projectA firewall enforces policy. A SIEM tells you what actually happened - allowed traffic, denied traffic, auth attempts, system events - searchable in one place instead of scattered across individual hosts. This project stands up Graylog as the log aggregation layer for the lab, starting with pfSense's own firewall logs.## Goals- [x] Stand up graylog01 as a new VM on the existing LAN segment- [x] Install and configure MongoDB (config data store)- [x] Install and configure OpenSearch (indexing/search engine)- [ ] Install and configure Graylog server- [ ] Access Graylog web UI, complete initial setup- [ ] Forward pfSense firewall logs to Graylog- [ ] Confirm real log events (allow/deny) searchable in Graylog dashboard- [ ] Narrow graylog01's firewall rule from temporary "Any" to least-privilege (HTTP/HTTPS only)## Lab architecture**Topology:**| Component | Role | Network ||---|---|---|| pfSense | Firewall / gateway | WAN: NAT - LAN: host-only || `theone` (Ubuntu Server, LibreNMS) | Monitored host | LAN (host-only) || `graylog01` (Ubuntu Server, Graylog SIEM) | Log aggregation target | LAN (host-only), 192.168.56.105 || Kali | Attack/test box | LAN (host-only) |`graylog01` specs: Ubuntu Server (64-bit), 4GB RAM, 2 CPU, 40GB dynamically allocated disk, single host-only network adapter (Intel PRO/1000 MT Desktop).## Current Status (as of 2026-07-15)**Working and confirmed:**- graylog01 VM built, networked, and reachable through pfSense- MongoDB 4.4 installed and running- OpenSearch 2.19.6 installed and running, responding to authenticated API calls**Interrupted / not yet confirmed:**- Graylog server package install was in progress when a host hardware failure (RAM) forced an emergency shutdown- `/etc/graylog/server/server.conf` has not been edited;
